\section{Executive Summary}

Crystal Forge is a full-stack research system for deciding \emph{how to simulate} and \emph{how to measure} correlated lattice models, with an explicit emphasis on superconductivity-motivated Hubbard workflows.

The project began as a small-lattice exact simulation engine for the spinful Fermi-Hubbard model and the transverse-field Ising model (TFIM). It then evolved into a layered decision system with four top-level responsibilities:

\begin{enumerate}[label=\arabic*.]
\item \textbf{Problem specification}: define a lattice many-body instance in a model-agnostic way.
\item \textbf{Regime routing}: decide whether the instance is still classically tractable or should escalate to a stronger quantum path.
\item \textbf{Measurement planning}: reduce quantum measurement overhead while preserving physically relevant observables.
\item \textbf{Scientific workflow packaging}: expose the resulting logic in a way that is meaningful to condensed matter and quantum computing users.
\end{enumerate}

The central lesson of the project is now clear:

\begin{quote}
Generic black-box routing and generic black-box measurement compression are weaker than physics-structured routing and physics-structured measurement planning.
\end{quote}

That lesson appears twice.

\begin{itemize}
\item For \textbf{CorrMap}, performance improved once the public routing task was reframed from a brittle three-way distinction to a binary classical-vs-quantum decision supported by intrinsic and physical tractability checks.
\item For \textbf{Adaptive QProbe}, performance improved only after the measurement problem was reframed around superconductivity-relevant channels and physically coherent decompositions of pairing and transport observables.
\end{itemize}

The current system should therefore be understood as a \textbf{superconductivity-first orchestration framework} rather than as a generic arbitrary-Pauli compression engine.

\subsection{Plain-English project statement}

For a non-specialist, the most accurate short description is:

\begin{quote}
Crystal Forge decides whether a lattice problem is still classically solvable, when a quantum path is justified, and how to measure the quantum result efficiently enough to recover superconductivity-relevant diagnostics.
\end{quote}

\subsection{Current headline results}

The strongest currently defended results in the repository are:

\begin{itemize}
\item \textbf{Binary CorrMap benchmark}: \(85.0\%\) overall accuracy on the corrected mixed-family benchmark; \(83.3\%\) on benchmark \(6\times 6\); \(86.7\%\) on held-out benchmark \(8\times 8\); \(90.0\%\) Hubbard accuracy; \(80.0\%\) TFIM accuracy.
\item \textbf{Adaptive superconductivity workflow}: on hard Hubbard superconductivity bundles in the fast heuristic benchmark, the safe fraction improves from \(21.9\%\) under naive channelization to \(41.8\%\) under transverse-signature decomposition at tolerance \(0.03\), then to \(55.7\%\) at tolerance \(0.05\), and \(67.1\%\) at tolerance \(0.08\).
\item \textbf{Recoverable hard-sector coverage}: under symmetry/orbit closure in the superconductivity workflow, recoverable hard-sector coverage reaches \(61.3\%\) at tolerance \(0.03\) and \(65.8\%\) at tolerances \(0.05\) and \(0.08\).
\end{itemize}

These numbers are not a claim that arbitrary frontier measurement planning has been solved. They are a claim that a physically structured superconductivity workflow yields \emph{substantial, benchmarked progress} in the hard sector where the generic adaptive planner failed.

\section{Project Thesis and Scientific Framing}

\subsection{Core thesis}

The project is built on the following thesis:

\begin{quote}
Small exact lattice solvers are best used as calibration systems for scalable routing and measurement policies, not as the final runtime engine.
\end{quote}

This has two consequences.

\begin{enumerate}[label=\arabic*.]
\item \textbf{Exact solvers define benchmark truth where available.}
\item \textbf{Deployed logic must depend only on quantities available at runtime on larger systems.}
\end{enumerate}

\subsection{Why superconductivity became the primary framing}

The most difficult part of the project turned out not to be exact diagonalization, but \emph{frontier measurement planning} for physically meaningful observables. In the generic operator-language framing, Adaptive QProbe underperformed in the hardest regime. The failure mode was especially severe for mixed, high-support operator bundles resembling Hubbard pairing and transport sectors.

That failure led to a more realistic scientific framing:

\begin{itemize}
\item researchers studying correlated materials do not usually care about arbitrary Pauli strings,
\item they care about charge, spin, transport, and pairing diagnostics,
\item superconductivity is governed by competition among those channels,
\item therefore the correct product surface is a \textbf{channelized measurement workflow}, not a raw operator soup.
\end{itemize}

This reframing is not merely a demo patch. It is closer to how a real user would structure a Hubbard measurement study.

\subsection{Research question}

The system now targets the following research question:

\begin{quote}
Can a software stack use small-system oracle information and physics-aware structure to decide when a strongly correlated lattice problem needs quantum treatment and how to allocate scarce quantum measurements toward superconductivity-relevant observables?
\end{quote}

\section{Problem Specification Layer}

\subsection{Motivation}

The first major architectural decision in the project was to separate:

\begin{itemize}
\item the \emph{mathematical problem},
\item the \emph{solver},
\item the \emph{observable vocabulary},
\item the \emph{routing and measurement decisions}.
\end{itemize}

This is implemented by the generic \verb|ProblemSpec| abstraction in \verb|backend/app/domain/problem_spec.py|.

\subsection{Structure}

The problem layer defines:

\begin{itemize}
\item \verb|LatticeSpec|: size, geometry, boundary conditions, site count, qubit count.
\item \verb|ParameterSpec|: model-parameter dictionary.
\item \verb|ProblemSpec|: model family, lattice, parameters, and metadata.
\end{itemize}

The convenience constructors are:
\[
\texttt{ProblemSpec.hubbard}(L_x,L_y,t,U,\mu),
\qquad
\texttt{ProblemSpec.tfim}(L_x,L_y,J,h,g).
\]

\subsection{Why this matters}

This abstraction is what makes it possible to use one frontend and one orchestration layer while supporting:

\begin{itemize}
\item exact solvers,
\item cheap classical solvers,
\item quantum escalation,
\item CorrMap routing,
\item QProbe planning,
\item family-specific physical interpretations.
\end{itemize}

\section{Physics Layer}

\subsection{Spinful Fermi-Hubbard model}

The spinful Fermi-Hubbard Hamiltonian is
\begin{equation}
H_{\mathrm{Hub}}
=
-t \sum_{\langle i,j\rangle}\sum_{\sigma\in\{\uparrow,\downarrow\}}
\left(c_{i\sigma}^{\dagger}c_{j\sigma}+c_{j\sigma}^{\dagger}c_{i\sigma}\right)
+U\sum_i n_{i\uparrow}n_{i\downarrow}
-\mu\sum_i(n_{i\uparrow}+n_{i\downarrow}).
\label{eq:hubbard}
\end{equation}

The terms have the standard physical interpretation:

\begin{itemize}
\item \(t\): hopping / kinetic scale,
\item \(U\): on-site interaction,
\item \(\mu\): chemical potential controlling filling and doping.
\end{itemize}

For the project, the Hubbard model is not just a toy correlated fermion system. It is the \textbf{superconductivity-facing family}. The observables and workflows we care about are motivated by:

\begin{itemize}
\item charge response,
\item antiferromagnetic order,
\item transport / kinetic channels,
\item singlet pairing coherence.
\end{itemize}

\subsection{TFIM}

The transverse-field Ising model used in the project is
\begin{equation}
H_{\mathrm{TFIM}}
=
-J \sum_{\langle i,j\rangle} Z_i Z_j
-h \sum_i X_i
-g \sum_i Z_i.
\label{eq:tfim}
\end{equation}

TFIM plays two roles:

\begin{itemize}
\item it validates that the routing stack is not Hubbard-only,
\item it provides a model family where a quantum escalation path is already implemented.
\end{itemize}

\subsection{Qubit mapping and exact linear algebra}

For Hubbard, the project uses a fermion-to-qubit mapping implemented in the backend physics layer. Exact diagonalization is then used on the resulting qubit Hamiltonian for small systems. TFIM uses a direct spin representation.

The exact solver has three jobs:

\begin{enumerate}[label=\arabic*.]
\item verify Hamiltonian construction,
\item benchmark cheap solvers,
\item produce oracle-quality measurement benchmarks when exact QProbe remains tractable.
\end{enumerate}

\section{Observable Layer}

\subsection{Shared principle}

The observable layer is the project-wide coordinate system. Every important subsystem consumes the same observable registry:

\begin{itemize}
\item exact solvers,
\item cheap solvers,
\item CorrMap,
\item exact QProbe,
\item bounded adaptive QProbe,
\item frontend reporting.
\end{itemize}

\subsection{Hubbard observables}

The current Hubbard registry includes:

\begin{itemize}
\item \(n\): average filling,
\item \(D\): double occupancy,
\item \(M_s^2\): staggered magnetization squared,
\item \(K\): kinetic / hopping signal,
\item \(C_{s,\max}\): long-range spin correlation,
\item \(\mathrm{Pair}_{nn}\): nearest-neighbor singlet pair density,
\item \(\mathrm{Pair}_{span}\): long-range singlet pair coherence.
\end{itemize}

These are not random diagnostics. They intentionally span the physically relevant superconductivity axes:

\begin{center}
\begin{tabular}{ll}
\toprule
Channel & Representative observables \\
\midrule
Charge & \(n, D\) \\
Spin / competing order & \(M_s^2, C_{s,\max}\) \\
Transport & \(K\) \\
Pairing & \(\mathrm{Pair}_{nn}, \mathrm{Pair}_{span}\) \\
\bottomrule
\end{tabular}
\end{center}

\subsection{TFIM observables}

The TFIM registry includes:

\begin{itemize}
\item \(M_z\),
\item \(M_x\),
\item \(ZZ_{nn}\),
\item \(M_{\mathrm{stag}}^2\),
\item \(Z_{\mathrm{span}}\).
\end{itemize}

These play the same structural role for TFIM that \(\{n,D,M_s^2,K,C_{s,\max},\mathrm{Pair}_{nn},\mathrm{Pair}_{span}\}\) play for Hubbard.

\section{Solver Layer}

\subsection{Exact solver}

Exact diagonalization is the oracle on small lattices. It is not the intended deployment path. Its role is:

\begin{itemize}
\item Hamiltonian validation,
\item small-system benchmark generation,
\item route-label and measurement-label supervision,
\item quality auditing.
\end{itemize}

\subsection{Cheap classical solvers}

The backend includes family-specific cheap solvers:

\begin{itemize}
\item unrestricted mean-field / classical Hubbard approximations,
\item TFIM mean-field approximations.
\end{itemize}

These are not assumed to be always correct. They are the input to CorrMap.

\subsection{Quantum escalation}

The TFIM family has a quantum escalation branch via VQE. This is important because it demonstrates the end-to-end architecture:

\[
\text{cheap solver}
\rightarrow
\text{CorrMap}
\rightarrow
\text{quantum branch if needed}
\rightarrow
\text{QProbe on the quantum result}.
\]

Hubbard does not yet have a production quantum branch in the repository, but the measurement-planning work is already structured around what a superconducting Hubbard quantum workflow would require.

\section{Materials and Regime Classification}

\subsection{What is being classified}

The project does not claim to classify real laboratory materials directly from first principles. The classification target is:

\begin{quote}
the computational regime of a lattice many-body instance: is it still in a classically scalable regime, or is it sufficiently correlated and difficult that a quantum/frontier treatment is justified?
\end{quote}

That is the correct interpretation of the materials/regime layer.

\subsection{Why the old three-way framing was replaced}

Early CorrMap experiments used a three-way route:

\[
\{\texttt{mean\_field},\texttt{scalable\_classical},\texttt{quantum\_frontier}\}.
\]

This turned out to be conceptually and empirically weaker:

\begin{itemize}
\item the \texttt{mean\_field} class was the least stable,
\item many ``bad'' predictions were actually harmless if both outputs stayed classical,
\item the real workflow question is not ``is mean field perfect?'' but ``is this still a classical problem at all?''
\end{itemize}

The public CorrMap route is therefore now binary:

\[
\{\texttt{classical\_scalable},\texttt{quantum\_frontier}\}.
\]

\section{CorrMap}

\subsection{CorrMap question}

The canonical CorrMap question is:

\begin{quote}
Given a cheap-solver output and scalable runtime diagnostics, should this problem stay on the classical side, or should it escalate toward a stronger quantum/frontier path?
\end{quote}

\subsection{CorrMap feature construction}

CorrMap starts from a family-aware cheap feature vector of dimension \(22\), then augments it with:

\begin{itemize}
\item intrinsic diagnostics computed on the target lattice,
\item physical tractability checks,
\item family-aware metadata.
\end{itemize}

The deployed augmented CorrMap feature vector has dimension \(49\).

\subsection{Intrinsic diagnostics}

The intrinsic diagnostics summarize the cheap solver's behavior across seeds, continuation, lattice transfer, and observable consistency. The key idea is that the deployment path must only use quantities available at runtime on larger lattices. This was a major methodological constraint throughout the project.

The intrinsic features include proxies for:

\begin{itemize}
\item residual and self-consistency stability,
\item seed variance,
\item hysteresis,
\item finite-size drift,
\item ansatz disagreement,
\item continuity under nearby parameter changes.
\end{itemize}

\subsection{Physics-aware tractability checks}

The strongest CorrMap improvement came when explicit physics was added rather than relying on pure oracle imitation. The current CorrMap uses first-class physical checks for:

\begin{itemize}
\item mean-field plausibility,
\item scalable classical plausibility,
\item frontier pressure,
\item sign/stoquasticity-style tractability proxies.
\end{itemize}

For Hubbard, these checks encode ideas such as:

\begin{itemize}
\item strong coupling and near half-filling increase frontier pressure,
\item doping and weak interactions increase classical plausibility,
\item factorization and low-correlation signatures increase mean-field plausibility.
\end{itemize}

For TFIM, they encode ideas such as:

\begin{itemize}
\item stoquasticity and field-dominated regimes favor classical plausibility,
\item proximity to the low-bias critical line increases frontier pressure.
\end{itemize}

\subsection{Training methodology}

CorrMap was trained under a deliberately staged split:

\begin{itemize}
\item small-lattice route supervision: \(2\times 2\) and \(2\times 3\),
\item intrinsic supervision: \(4\times 4\) and \(6\times 6\),
\item model selection / calibration benchmark: corrected benchmark \(6\times 6\),
\item held-out test benchmark: corrected benchmark \(8\times 8\).
\end{itemize}

The key ethical claim is that the public \(8\times 8\) numbers are \emph{not} trained on \(8\times 8\) routing data.

\subsection{Benchmark construction}

To avoid evaluating CorrMap only against its own internal label system, the project built a corrected independent benchmark with:

\begin{itemize}
\item balanced hand-curated regime points,
\item family coverage across Hubbard and TFIM,
\item explicit rationale,
\item corrected TFIM boundary cases in the refined \verb|v2| benchmark path.
\end{itemize}

\subsection{CorrMap results}

The primary public CorrMap result is the binary benchmark.

\begin{center}
\begin{tabular}{lc}
\toprule
Metric & Value \\
\midrule
Overall corrected benchmark accuracy & \(85.0\%\) \\
Benchmark \(6\times 6\) accuracy & \(83.3\%\) \\
Held-out benchmark \(8\times 8\) accuracy & \(86.7\%\) \\
Hubbard accuracy & \(90.0\%\) \\
TFIM accuracy & \(80.0\%\) \\
\bottomrule
\end{tabular}
\end{center}

This is the strongest CorrMap result in the repository. It is strong enough to present as a working prototype of regime routing under the revised methodology.

\subsection{What CorrMap does and does not claim}

The correct public claim is:

\begin{quote}
CorrMap works as a binary classical-vs-quantum regime router that combines small-lattice supervision, larger-lattice intrinsic diagnostics, and physics-aware tractability checks.
\end{quote}

The incorrect claim would be:

\begin{quote}
CorrMap is a theorem-like certification oracle for large lattices.
\end{quote}

It is not that. It is a benchmarked routing prototype.

\section{QProbe: Exact Measurement Planning}

\subsection{Exact QProbe question}

QProbe asks:

\begin{quote}
What is the cheapest measurement plan that recovers the requested observables within a specified tolerance?
\end{quote}

\subsection{Representation}

Each target observable is compiled to a \verb|SparsePauliOp|. The measurement layer then:

\begin{enumerate}[label=\arabic*.]
\item decomposes the operator into Pauli terms,
\item groups compatible terms into measurement families,
\item merges families across requested targets where possible,
\item searches for the minimum-cost plan satisfying the tolerance target.
\end{enumerate}

\subsection{Exact QProbe}

Exact QProbe is combinatorial. Its role in the project is:

\begin{itemize}
\item oracle planning on tractable requests,
\item supervision / auditing for adaptive policies,
\item benchmarking the cost of compression.
\end{itemize}

The critical lesson from the project is that exact QProbe remains invaluable even when it is too expensive to use everywhere, because it defines the \emph{overlap} and \emph{edge} calibration zones.

\section{Adaptive QProbe: Generic Phase}

\subsection{Original adaptive goal}

Adaptive QProbe was originally intended to be the scalable runtime version of exact QProbe:

\begin{quote}
stop as early as possible while staying close to exact optimal measurement cost.
\end{quote}

\subsection{Why the original generic version underperformed}

The generic adaptive planner was pushed through several versions:

\begin{itemize}
\item learned stop classifiers,
\item calibrated selective classifiers,
\item stepwise adaptive models,
\item bounded-search optimization with DP lower bounds and branch-and-bound heuristics.
\end{itemize}

The important negative result is that \emph{generic frontier planning on arbitrary hard operator strings did not become good enough}. In particular:

\begin{itemize}
\item the learned stop versions were too brittle,
\item the bounded optimization path improved overlap/edge behavior,
\item but the frontier zone remained weak for hard mixed bundles.
\end{itemize}

\subsection{Overlap, edge, and frontier zones}

The project now evaluates adaptive planning in a three-zone stack:

\begin{itemize}
\item \textbf{Overlap}: exact QProbe is cheap enough to run comfortably.
\item \textbf{Edge}: exact QProbe is expensive but still feasible for calibration.
\item \textbf{Frontier}: exact QProbe is no longer practical; planner quality is measured against a certified lower bound or related heuristics.
\end{itemize}

This stack is scientifically important because it is the only coherent way to justify any scalable adaptive planner.

\section{Why the Project Pivoted to Superconductivity}

\subsection{Diagnosis of the hard-sector failure}

The hardest failures in adaptive planning were not random. They were concentrated in operator bundles that looked like:

\begin{itemize}
\item mixed charge/spin/transport/pairing requests,
\item high support,
\item strong basis conflict,
\item many incompatible measurement families.
\end{itemize}

Those are exactly the operator bundles that arise in superconductivity-facing Hubbard studies. This led to the central pivot:

\begin{quote}
Do not ask Adaptive QProbe to solve arbitrary Pauli soups. Ask it to solve a structured superconductivity workflow.
\end{quote}

\subsection{Superconductivity workflow channels}

The project therefore restructured hard Hubbard measurements into four channels:

\begin{center}
\begin{tabular}{ll}
\toprule
Channel & Targets \\
\midrule
Charge & \(n, D\) \\
Spin & \(M_s^2, C_{s,\max}\) \\
Transport & \(K\) \\
Pairing & \(\mathrm{Pair}_{nn}, \mathrm{Pair}_{span}\) \\
\bottomrule
\end{tabular}
\end{center}

\subsection{Physical motivation}

This channelization is not arbitrary UI design. It reflects how a researcher would actually analyze a correlated superconductivity problem:

\begin{itemize}
\item charge tells you filling and local interaction response,
\item spin captures competing antiferromagnetic order,
\item transport captures kinetic coherence,
\item pairing captures superconducting order and pair coherence.
\end{itemize}

\section{Adaptive QProbe: Superconductivity Workflow}

\subsection{First workflow step: channelization}

The first improvement was to split hard bundles into channels and plan those separately. This alone did not solve the hard sector, but it established a meaningful scientific problem decomposition.

\subsection{Second workflow step: transverse-signature decomposition}

The key practical improvement came from decomposing pairing and transport observables by a coarse basis signature:

\[
X,Y \mapsto T,\qquad Z\mapsto Z,\qquad I\mapsto I.
\]

This groups many raw Pauli-measurement families into a smaller number of physically coherent transverse-vs-longitudinal subtargets. For example, on a representative doped Hubbard point:

\begin{itemize}
\item \(K\) collapses from \(4\) raw groups to \(2\) transverse-signature classes,
\item \(\mathrm{Pair}_{nn}\) collapses from \(17\) raw groups to \(3\) signature classes,
\item \(\mathrm{Pair}_{span}\) collapses from \(32\) raw groups to \(4\) signature classes.
\end{itemize}

This was the first modification that materially improved hard-sector adaptive coverage.

\subsection{Third workflow step: symmetry/orbit closure}

The next workflow improvement was to group decomposed subtargets into orbit classes using simple signature counts:

\[
\text{orbit key} = (N_I, N_Z, N_T).
\]

For a CS audience, this should be interpreted as \emph{equivalence-class closure}. If a difficult target family decomposes into several structurally equivalent representatives and at least one representative is safely handled, the orbit becomes \emph{recoverable} even if not every raw subtarget is independently safe.

\subsection{Safe versus recoverable}

This distinction is essential:

\begin{itemize}
\item \textbf{Safe}: the subproblem was solved directly by adaptive planning and passed heuristic validation as-is.
\item \textbf{Recoverable}: the workflow can reconstruct or stand in for a subproblem using redundancy / equivalence among successful representatives.
\end{itemize}

In computer science terms:

\begin{itemize}
\item \textbf{safe} is direct pass rate,
\item \textbf{recoverable} is effective coverage after structured reuse of equivalent cases.
\end{itemize}

\subsection{Benchmark methodology for the superconductivity workflow}

The fast hard-sector benchmark used in the last project phase has the following properties:

\begin{itemize}
\item focus only on hard Hubbard superconductivity bundles,
\item no exact subset search inside the benchmark loop,
\item compare monolithic bounded adaptive, channelized adaptive, decomposed adaptive, and workflow-level adaptive,
\item use heuristic validation quantities already available in the planner,
\item sweep tolerance to study the stability/usefulness tradeoff.
\end{itemize}

The hard superconductivity bundles used in quick mode are:

\begin{itemize}
\item \texttt{pair\_sector}: \((\mathrm{Pair}_{nn}, \mathrm{Pair}_{span})\),
\item \texttt{transport\_pairing}: \((K,\mathrm{Pair}_{nn},\mathrm{Pair}_{span})\),
\item \texttt{competing\_orders}: \((n,M_s^2,\mathrm{Pair}_{nn},\mathrm{Pair}_{span})\),
\item \texttt{superconductor\_panel}: \((D,K,M_s^2,\mathrm{Pair}_{nn},\mathrm{Pair}_{span})\).
\end{itemize}

These are intentionally hard because they mix the channels that matter most for superconductivity.

\subsection{Superconductivity workflow results}

The benchmarked progression is:

\begin{center}
\begin{tabular}{lccc}
\toprule
Method / metric & \(0.03\) tol & \(0.05\) tol & \(0.08\) tol \\
\midrule
Channelized safe fraction & \(0.21875\) & \(0.29167\) & \(0.29167\) \\
Decomposed safe fraction & \(0.41775\) & \(0.55700\) & \(0.670996\) \\
Workflow safe fraction & \(0.41775\) & \(0.55700\) & \(0.670996\) \\
Workflow recoverable fraction & \(0.6125\) & \(0.65833\) & \(0.65833\) \\
\bottomrule
\end{tabular}
\end{center}

Interpretation:

\begin{itemize}
\item At tight tolerance \(0.03\), the physically structured workflow nearly doubles direct hard-sector safe coverage relative to naive channelization.
\item At moderate tolerance \(0.05\), the hard-sector safe fraction crosses \(55\%\).
\item At tolerance \(0.08\), the hard-sector safe fraction reaches roughly \(67\%\), which is close to the ``mostly works'' threshold for a hackathon demo.
\item Recoverable coverage exceeds \(60\%\) even at tight tolerance because some unresolved subtargets belong to orbits with successful representatives.
\end{itemize}

\subsection{What the workflow still does not solve}

It is important to state what remains open:

\begin{itemize}
\item monolithic hard superconductivity bundles are still not solved directly,
\item direct \(\mathrm{Pair}_{nn}\) and \(K\)-heavy subtargets remain the most difficult,
\item the current workflow is a strong \emph{partial solution}, not a full frontier measurement solution.
\end{itemize}

\section{Relationship to the Literature}

\subsection{What ideas we borrowed implicitly or explicitly}

The final superconductivity framing is consistent with several ideas from the measurement-reduction literature:

\begin{itemize}
\item fast partitioning of Pauli strings into commuting families,
\item fermionic partial tomography / fermionic shadow ideas,
\item low-rank or reduced-density-matrix style recovery of structured observables,
\item measurement of physically coherent sectors rather than arbitrary operator soups.
\end{itemize}

\subsection{Why those ideas matter here}

The project's actual adaptive failure mode was not that the planner could not handle \emph{any} hard observable. It was that it was asked to solve too many incompatible hard observables simultaneously. The literature suggests that the right response is not more black-box stopping heuristics, but more structure:

\begin{itemize}
\item partition more intelligently,
\item exploit symmetry and equivalence,
\item reconstruct from structured low-dimensional sectors when possible.
\end{itemize}

That is exactly the direction the superconductivity workflow now takes.

\section{Benchmarking Methodology}

\subsection{CorrMap benchmarking}

CorrMap uses a corrected, independently curated regime benchmark with:

\begin{itemize}
\item family balance,
\item explicit regime rationale,
\item separate \(6\times 6\) calibration and \(8\times 8\) held-out evaluation.
\end{itemize}

Metrics:

\begin{itemize}
\item overall accuracy,
\item family-specific accuracy,
\item classwise precision, recall, and F1 where relevant.
\end{itemize}

\subsection{QProbe benchmarking}

QProbe benchmarking is divided into:

\begin{itemize}
\item \textbf{exact overlap}: compare directly to exact QProbe where tractable,
\item \textbf{edge}: compare to exact regret on still-solvable hard requests,
\item \textbf{frontier}: compare to certified lower bounds or heuristic workflow metrics where exact is too expensive.
\end{itemize}

\subsection{Why heuristics were acceptable at the end}

For the final hackathon-facing superconductivity workflow, heuristic checks were deliberately used because:

\begin{itemize}
\item full exact subset search on the hardest bundles is too slow for rapid iteration,
\item the objective was to determine whether physically structured workflow design can make meaningful progress,
\item this was a systems-design and framing question as much as an optimization question.
\end{itemize}

This is acceptable because the document clearly distinguishes:

\begin{itemize}
\item exact benchmarks where available,
\item heuristic hard-sector workflow benchmarks where exact search is impractical.
\end{itemize}

\section{Frontend and Product Framing}

\subsection{Correct public demo framing}

The correct demo framing is:

\begin{enumerate}[label=\arabic*.]
\item user specifies a lattice problem,
\item cheap solver produces a first-pass state,
\item CorrMap decides whether the problem remains classical or should escalate,
\item if a quantum path is used, QProbe / Adaptive QProbe builds a superconductivity-aware measurement workflow,
\item the UI explains what parts of the superconductivity panel are directly safe and what parts are only recoverable through structured decomposition.
\end{enumerate}

\subsection{What the demo should not claim}

The demo should not claim:

\begin{itemize}
\item universal frontier measurement optimality,
\item arbitrary Pauli-string compression at scale,
\item theorem-level correctness on large materials systems.
\end{itemize}

\subsection{What the demo can claim}

The demo \emph{can} claim:

\begin{itemize}
\item binary classical-vs-quantum routing works well on the corrected benchmark,
\item exact QProbe works in the tractable regime,
\item generic adaptive planning fails on hard mixed superconductivity bundles,
\item a superconductivity-aware workflow materially improves hard-sector coverage,
\item physically structured decomposition is a better route than generic black-box planning for these observables.
\end{itemize}

The important point is not that it is finished; it is that it now demonstrates a credible path forward with benchmark movement in the correct physical regime.

\section{Final Position}

The final position of the project is:

\begin{enumerate}[label=\arabic*.]
\item \textbf{CorrMap succeeded under the correct framing.}
Binary classical-vs-quantum routing with physics-aware diagnostics is a strong result.

\item \textbf{Exact QProbe succeeded in the tractable regime.}
It remains the oracle for calibration and evaluation.

\item \textbf{Generic adaptive frontier planning did not fully succeed.}
This negative result was important and shaped the scientific pivot.

\item \textbf{The superconductivity workflow rescued the adaptive story enough to matter.}
Physical channelization, transverse-signature decomposition, and orbit closure turned a failing hard sector into a partially working one with meaningful coverage.
\end{enumerate}

This is therefore a coherent project with a real scientific arc:

\begin{quote}
when generic black-box scaling failed, physics-aware restructuring made progress exactly where the application cared most.
\end{quote}

That is a strong hackathon result, a credible research prototype, and a realistic foundation for future work.
